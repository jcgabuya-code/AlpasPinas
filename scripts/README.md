# scripts/

One-off developer/debugging scripts. These are **not** part of the app build or
an automated test suite — they're ad-hoc Node scripts (mostly Playwright and
Supabase/Sheets checks) used to poke at the running app or backends during
development.

Run them from the project root, e.g.:

```bash
node scripts/test-admin.mjs
```

Most Playwright scripts expect the dev server running (`npm run dev`, default
`http://localhost:5173`).

## Contents

- `screenshot_mobile.mjs` — capture a mobile-viewport screenshot of the app.
- `test-admin*.mjs` — drive the `/admin` page (PIN unlock, approve flow, etc.).
- `test-approve*.mjs` — exercise the booking approval flow / RPC variations.
- `test-signups*.mjs`, `test-logout.mjs`, `test-direct.mjs`, `test-simple.mjs`,
  `test-wait-longer.mjs`, `test-check-error.mjs` — assorted flow checks.
- `check-*.mjs` — inspect backend state (current status, events, sheet rows).
- `add-test-booking.mjs` — seed a test booking.

Screenshot output (`*.png`) generated by these scripts is gitignored.
